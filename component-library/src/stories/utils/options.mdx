# options
## [src/lib/utils/options](https://github.com/coxco96/mapbox-svelte-components/tree/main/component-library/src/lib/utils/options)
### default options
This folder holds the default options exports for the default options for options not supplied by the user in `<MapContainer/>`:
- [mapOpts](https://github.com/coxco96/mapbox-svelte-components/blob/main/component-library/src/lib/utils/options/map-options.js): 
    - `import { options } from '$lib/utils/options/map-options.js'`
- [geocoderOpts](https://github.com/coxco96/mapbox-svelte-components/blob/main/component-library/src/lib/utils/options/geocoder-options.js): 
    - `import { options } from $lib/utilsoptions/geocoder-options.js` 
       
- [data](https://github.com/coxco96/mapbox-svelte-components/blob/main/component-library/src/lib/utils/options/geojson-options.js):  
    - `import { sourceOptions } from '$lib/utils/geojson-options.js'`

### style options
This folder also holds [object of map styles](https://github.com/coxco96/mapbox-svelte-components/blob/main/component-library/src/lib/utils/options/map-style-options.js) for easier reference to set style in `mapOpts`:
- `import { styles } from '$lib/utils/options/map-style-options.js'`

It comes with the Mapbox-owned styles. The user may edit this file to include references to their own style URLs.
With the file imported, Mapbox's 'light' style can be specified like this:
            - `<MapContainer mapOpts={{style: styles.light}}/>`
            
Whereas without the file, the full URL must be typed out, like this:
            - `<MapContainer mapOpts={{style: 'mapbox://styles/mapbox/light-v10'}}/>`
