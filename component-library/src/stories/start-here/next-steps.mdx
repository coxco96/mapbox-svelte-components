# what's next?
---
- ### ideas for accessibility and equitable data experiences
        - write scripts to analyze map options, generating a "for humans" summary of the map's style and location and boundaries, to be sent to a screen-reader only hidden dom element on render
        - write scripts to analyze any accepted data type for this map (so far only geojson) to generate a summary of interesting bits about the data, to be sent to a screen-reader only hidden dom element on render
        - create reactive declarations to re-run these scripts to announce changes in map or data displayed
        - consider reduce motion preferences (rotating globe needs option to stop rotation)
        - automatically generate a data table from source data and make available as hidden node for screen readers
